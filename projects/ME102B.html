Automated Pill Dispenser

Mechatronic system designed to improve medication adherence through automated, reliable pill dispensing.

Mechatronics Mechanical Design Embedded Systems Prototyping

Overview

Designed and built an automated pill dispensing system targeting medication adherence challenges for users with cognitive or physical limitations. The system integrates mechanical actuation, control logic, and embedded hardware into a unified mechanism.

Problem

Medication mismanagement is a common issue for individuals with memory decline or physical limitations, often leading to missed or incorrect dosages. This project aimed to develop a compact mechanical solution that automates daily pill dispensing with minimal user interaction.

Design Approach

  • Rack-and-pinion mechanism used for controlled linear motion of pill caddy
  • Single-motor architecture to reduce system complexity and failure points
  • Constrained rail system introduced to improve alignment and motion stability
  • Front plate redesigned with bushings to support cantilevered loading
Iterated through 3 full CAD revisions to improve mechanical reliability and reduce jamming during actuation.

Key Engineering Decisions

  • Replaced free-floating rack with guided aluminum rail system for repeatable motion
  • Added bushings to reduce friction and support shaft loading
  • Shifted to single-controller architecture to avoid inter-system synchronization issues
  • Optimized assembly using wood dowels and 4-40 fasteners for manufacturability

System Notes

Due to the absence of encoders, system tuning was performed empirically using PWM adjustments and iterative testing. While this reduced absolute positioning precision, it significantly simplified system integration and improved robustness under project constraints.

Results

Achieved consistent automated dispensing with stable mechanical motion and repeatable positioning across ~50+ test cycles in final prototype.

The final system demonstrated reliable operation under repeated testing conditions with no critical jamming events in the final iteration.
